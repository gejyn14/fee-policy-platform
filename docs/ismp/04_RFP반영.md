# 산출물 ④ RFP 반영

- 대상 과제: 수수료 체계 재설계 (fee 도메인)
- 기준: 기술설계서 v1.7 + 실 구현
- 범위: 사업의 발주 범위·제약·평가 기준·규모 근거를 정의합니다. **확정 설계를 못박아 벤더의 가치 부가 여지를 제거하지 않습니다.** 성과 목표와 제약을 정의하고 물리 설계·사이징·구현은 벤더 과업으로 남깁니다.

## 1. 사업 범위 (벤더 구축 대상)

- 기본·이벤트·협의 3계층을 통합한 공통 정책 모델과 상태 수명주기 구축. (요구: FR-POL-01~04)
- 상품군별 비대칭 조회키(조회구분 축 포함)와 통합 우선순위 랭킹·자격 게이트 구축. (FR-KEY-01~03, FR-RANK-01~03)
- 배정판(계좌별 우대 승자) 사전 산출·3층 무효화·배정 이력 구축. (FR-BIND-01~04)
- 협의 신청·자격 자동판정·요청 단위 승인·조건 평가/연장 업무 기능 구축. (FR-NEG-01~02)
- 등록 시점 지배관계·역마진 검증 및 사전 시뮬레이션 구축. (FR-DOM-01~02)
- 요율표 평가(정률·정액·구간표·최소수수료)와 체결 시 요율 조회·결정 추적 구축. (FR-CALC-01, FR-LEDGER-01, FR-TRACE-01)
- 계좌·상품·요율표·정책·자격기준 마스터 및 배치 운영·관측 구축. (FR-MST-01, NFR-OPS-01)

## 2. 요구 참조

- 기능 요구: `01_요구사항정의서.md` 1장 FR 전체.
- 비기능 요구(원칙): 동 문서 2장 NFR 전체. 정량 목표는 4장·부록 A로 벤더 제안·협의 시 확정.
- 아키텍처 제약 원칙: `03_아키텍처방향.md` 결정 A~H.

## 3. 제약 사항 (벤더 준수)

- **원장 결합도**: 원장은 배정판 조회 계약(한 경로)만 알고 정책 평가 로직에는 의존하지 않음. 계약은 하위호환을 원칙으로 함. (NFR-COMPAT-01)
- **연계 방식 중립**: 직접 DB 쓰기/EAI 등 특정 연계 방식에 종속되지 않는 계약 구조 유지. (원장 실제 방식은 A-07 확정)
- **물리 스택 미확정**: 원장 실제 기술 스택·DBMS 확인 후 물리 설계 확정(참조 구현은 PostgreSQL, 물리 설계 문서는 Oracle 기본값 가정이며 어느 쪽도 확정 아님). → A-07.
- **축 배치 원칙 준수**: 매체=키, 세션·매수/매도 등 값 차이=요율표 내부(결정 C).
- **지배관계 전제**: 우대 정책은 기준 요율을 전 가격대에서 지배해야 하며, 교차 구간(가격대별 우열 역전) 동시 지원 여부는 확정 대상. → A-04.
- **배정판 저장 원칙**: 우대분만 저장, 미스=기본 직접 해석(전 계좌 공통·결정적). (결정 F)
- **범위 밖(현 단계 명시)**: 인증·권한, 실 원장 실연계, 운영 배포 인프라, 계좌·종목 마스터 동기화 배치는 별도 확정 대상. → A-08·A-11.

## 4. 평가 기준 (제안 심사 관점)

- **규모 대응 설계 역량**: 계좌×품목 전량 전개를 회피하는 배정판형 설계·구현 경험, 우대분만 저장·미스 정상 경로 설계.
- **정합성·안전망**: 3층 산출(증분/일 delta/전체 재산출) 및 전체 재산출 대조를 통한 정합성 보증 방안.
- **원장 연계 안전성**: 조회 계약 하위호환·shadow-run/dual-write 등 무중단 전환 절차.
- **대규모 판 운영 방안**: 억~수십억 규모의 배치 산출·적재 전략과 다중 인스턴스 정합성 대응.
- **성능·가용성 목표 달성 방안**: 정량 목표(부록 A) 충족 근거 및 열화 모드 대응.
- **추적성**: 배정 이력·결정 추적·조건 평가 이력의 감사 대응 설계.

## 5. 규모 근거 (제안 규모 산정용)

- 정책 저장 규모는 계좌 총량과 무관(요율표·활성 이벤트·협의 편입 건수 규모). (설계서 1장)
- 값비싼 축은 계좌×조회좌표(배정판)이며, **우대분만 저장**하여 우대 건수 규모로 축소됨. 주식 조회키는 품목 붕괴로 상수 규모, 파생은 품목 유지. (결정 A·B·F)
- 배정판 점조회는 인덱스 구조상 행 수와 무관하게 준실시간(참조 구현 로컬 실측: 2억 행에서도 점조회 중앙값 0.3ms 내외, 상세는 `05_부록_실현가능성증빙.md`). **실환경 정량 규모(계좌 수, 품목 수, 체결 피크 TPS, 데이터 보존기간, 상위 규모 실측)** → **【확인 필요】** 부록 A-01·A-12.
