# 산출물 ② 기능분해

- 대상 과제: 수수료 체계 재설계 (fee 도메인)
- 기준: 기술설계서 v1.7 + 실 구현
- 범위: 기능을 계층(대>중>소)으로 나열하고 소기능마다 1~3행 설명과 요구 ID를 참조합니다. 처리 로직·시퀀스·API 스펙은 기재하지 않습니다(구축 귀속).

## 1. 정책 관리
- **1.1 정책 등록(기안)**: 기본정보·적용범위·요율표·대상·시뮬레이션을 입력해 정책을 기안합니다. (FR-POL-01, FR-POL-03, FR-POL-04)
- **1.2 요율표 구성**: 구성요소(자사·유관기관·세금)와 방식(정률·정액·구간표)·부담주체·통화를 정의합니다. (FR-POL-01, FR-CALC-01)
- **1.3 정책 검증**: 상신 전 지배관계·역마진을 판정하고 시뮬레이션(대상 계좌·예상 감면)을 산출합니다. (FR-DOM-01, FR-DOM-02)
- **1.4 승인/반려**: 결재자가 검증 결과를 확인해 승인 또는 반려하고, 승인 시 배정판 증분을 반영합니다. (FR-POL-02, FR-BIND-03)
- **1.5 수명주기 전이**: 기간 도래·만료·조기종료에 따라 활성/종료 상태를 전이합니다. (FR-POL-02)

## 2. 협의수수료
- **2.1 협의 신청**: 계좌 단위로 적용범위·요율표·대상 계좌를 입력해 협의를 신청합니다. (FR-NEG-01)
- **2.2 자격 자동판정**: 자산군 표준 기준(자격기준)으로 계좌별 충족을 판정하고 미충족은 영업 예외로 표시합니다. (FR-NEG-01)
- **2.3 협의 승인/반려**: 요청번호 묶음 단위로 승인/반려하고 승인 시 활성화·해당 계좌 배정판 증분을 반영합니다. (FR-NEG-01, FR-BIND-03)
- **2.4 연장 대상 산출·연장**: 그룹(주식형=상품군, 파생=품목) 단위로 재평가해 유지/탈락을 분류하고 연장합니다. (FR-NEG-02)
- **2.5 조건 평가 이력**: 6개월 지표를 임계값과 비교한 평가 결과를 이력화합니다. (FR-NEG-02, NFR-AUDIT-01)

## 3. 요율 우선순위 엔진
- **3.1 조회키 구성**: 상품군별 입도(주식=조회구분 단위, 파생=품목 포함)로 체결 컨텍스트에서 조회키를 조립합니다. (FR-KEY-01, FR-KEY-02)
- **3.2 통합 랭킹 산정**: 기본·이벤트·협의를 하나의 요율 순위로 정렬합니다. (FR-RANK-01, FR-RANK-02)
- **3.3 자격 게이트 판정**: 계좌별 대상·기간 자격으로 후보를 걸러 첫 통과 정책을 승자로 확정합니다. (FR-RANK-03)
- **3.4 수수료 금액 산출**: 승자 요율표를 체결가·수량에 적용해 고객부과/회사부담 금액을 계산합니다. (FR-CALC-01)

## 4. 배정판 산출·무효화
- **4.1 전체 재산출(일배치)**: 활성 정책으로 계좌×조회좌표 배정판을 재유도하여 정합성 진본을 확정합니다. (FR-BIND-01, FR-BIND-03, NFR-CONSIST-01)
- **4.2 일 delta 배치**: 날짜 경과로 승자가 바뀔 수 있는 계좌만 재산출합니다. (FR-BIND-03)
- **4.3 수시 증분**: 정책 승인/종료·협의 승인/연장·이벤트 신청·휴면복귀 계기별 영향 계좌만 갱신합니다. (FR-BIND-03)
- **4.4 우대분 저장·미스 처리**: 우대 승자만 저장하고, 미저장 셀은 원장이 기본수수료로 처리합니다. (FR-BIND-02, FR-LEDGER-01)
- **4.5 배정 이력 적재**: 배정 변경(이전→이후)을 계기·사유와 함께 기록합니다. (FR-BIND-04, NFR-AUDIT-01)
- **4.6 대규모 판 처리 전략**: 억~수십억 규모에서 우선순위 계산 시점·산출 방식을 규모에 맞게 조정합니다. (FR-BIND-03, NFR-SCALE-01)

## 5. 대상 편입
- **5.1 신청형/가입형 편입**: 신청·가입 시 계좌 편입내역을 생성해 대상으로 편입합니다. (FR-POL-03)
- **5.2 휴면복귀 편입**: 휴면복귀 감지 신호로 대상에 자동 편입합니다. (FR-POL-03)
- **5.3 타겟추출형 편입**: 지정 계좌 목록(없으면 전체)으로 대상을 판정합니다. (FR-POL-03)

## 6. 검증
- **6.1 지배관계 검증**: 우대 요율표가 기준 요율표를 전 가격대에서 지배하는지 검사합니다. (FR-DOM-01)
- **6.2 역마진 판정**: 회사부담이 자사 수취를 초과하는지 판정·경고합니다. (FR-DOM-02)
- **6.3 사전 시뮬레이션**: 대상 계좌 수·예상 감면액을 상신 전 산출합니다. (FR-DOM-01)

## 7. 원장 연계·결정 추적
- **7.1 체결 시 요율 조회**: 원장이 체결 정보로 배정판을 조회하고 미스 시 기본 요율을 직접 해석합니다. (FR-LEDGER-01)
- **7.2 결정 추적**: 후보 정책·자격 게이트·탈락 사유·배정판 히트를 제시합니다. (FR-TRACE-01)

## 8. 마스터·운영
- **8.1 마스터 관리**: 계좌·상품(품목)·요율표·정책·자격기준 마스터를 관리합니다. (FR-MST-01)
- **8.2 셀 유니버스 범위**: 계좌가 개설한 상품군만 배정판 산출 대상으로 전개합니다. (FR-MST-01, NFR-SCALE-01)
- **8.3 배치 운영·관측**: 배치 실행 결과(건수·소요)와 배정 변경 현황을 관측합니다. (NFR-OPS-01)
