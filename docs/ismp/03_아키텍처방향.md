# 산출물 ③ 아키텍처 방향

- 대상 과제: 수수료 체계 재설계 (fee 도메인)
- 기준: 기술설계서 v1.7 + 실 구현(참조 구현으로 실현 가능성 검증됨, 부록 05 참조)
- 범위: 핵심 결정마다 근거·대안·리스크를 기술합니다. 물리 배치도·사이징·샤드 수·구현 알고리즘은 구축 단계 귀속이므로 제외합니다.

## 결정 A. 정책은 저장, 요율은 배정판으로 사전 산출 (resolve→배정판)
- **결정**: 계좌×품목 전량 사전 전개를 폐기하고, 정책(규칙·요율표·편입내역)만 저장합니다. 계좌별 승자는 배치가 **배정판**으로 사전 산출하며, 원장은 체결 시 배정판에서 한 줄만 조회합니다.
- **근거**: 전량 전개 시 계좌×품목 조합이 수백억 건으로 폭발해 저장·재계산이 불가합니다(설계서 1장). 배정판은 실제 우대 셀만 담아 규모를 억제하고, 원장은 정책·우선순위를 알 필요가 없습니다.
- **대안**: 체결 시점 실시간 해석(계산 후 캐시). → 승인 시점에 순위가 확정되는 불변식(가격 무관 승자)을 활용하면 사전 산출이 더 단순·결정적이므로 배정판형 채택. 전량 pre-binding은 규모상 실현 불가하여 기각.
- **리스크**: 배정판 산출 배치의 규모·시간(대규모 판)과 증분 누락 시 정합성이 관건 → 결정 E(3층 배치)로 대응.

## 결정 B. 조회키 입도의 상품군별 비대칭 + 조회구분 축
- **결정**: 조회키 입도를 상품군별로 다르게 둡니다. 주식군은 품목 차원을 붕괴(조회구분 단위), 파생군은 품목 차원을 유지합니다. 여기에 **조회구분**(선물/옵션/주식/ETF/금) 축을 두어 같은 상품군 내 상이한 요율을 분리합니다.
- **근거**: 카디널리티가 가장 큰 지점(주식=수천 종목 × 대량 계좌)에서 품목을 제거하는 것이 규모 억제의 핵심입니다(설계서 1·3장). 조회구분으로 선물/옵션·주식/ETF의 승자 경쟁을 올바르게 분리합니다.
- **대안**: 전 상품군 획일 키(품목 포함) 또는 증권유형·세제·개별종목 획일 결합. → 주식에서 조합 폭발을 유발하는 안티패턴이라 기각.
- **리스크**: 상품군별 조회구분·세션 유효값 매핑이 확정되어야 합니다. → 구축 시 확정.

## 결정 C. 축 배치 원칙: 매체는 판의 키, 값 차이는 요율표 내부
- **결정**: 정책 축을 "그 축에 따라 이기는 요율표가 달라지면 조회키(판의 키), 같은 요율표 안에서 값만 다르면 요율표 내부 항목"으로 배치합니다. 매체(채널)는 키, 매수/매도 등은 요율표 내부로 둡니다.
- **근거**: 매체 한정 이벤트는 매체에 따라 승자 정책 자체가 달라지므로 키로 분리해야 근거 추적이 오염되지 않습니다(설계서 3.3).
- **대안**: 세션·매체를 모두 요율표 내부로 흡수. → 승자가 달라지는 축을 내부로 밀면 인공 합성 요율표가 필요해 오류 여지가 크므로 기각.
- **리스크**: 축별 취급 차이가 조회 계약·셀 전개에 일관 반영되어야 합니다. → 구축 시 확정.

## 결정 D. 통합 우선순위 랭킹 (협의 무조건 우선 아님)
- **결정**: 기본·이벤트·협의를 하나의 요율 순위에 올려 경쟁시킵니다. 정렬은 ①요율 최저 → ②동률 시 계층(협의>이벤트>기본) → ③적용범위 구체성 → ④정책ID입니다. **협의가 무조건 우선이 아니며, 더 저렴한 이벤트가 있으면 이벤트가 승리**합니다(고객 유리 원칙). 계좌 적용은 편입내역(활성·유효기간)으로 판정합니다.
- **근거**: 같은 조회키 안에서 승자는 가격과 무관하다는 불변식(정률=요율, 정액=정액, 우대는 기본을 전 구간 지배)에 근거해 한 번의 순위가 전 가격대에서 성립합니다(설계서 5장). 협의를 오버레이가 아닌 규칙으로 일원화해 경로를 단일화합니다.
- **대안**: 협의를 별도 최우선 계층으로 승격(오버레이). → 이벤트가 더 저렴한 경우 고객에게 불리해지고 경로가 이원화되므로 기각(과거 v0.8 오버레이 모델 폐기).
- **리스크**: "협의는 계약상 반드시 적용" 같은 업무 요구가 있으면 tie-break이 아니라 계층 승격이 필요할 수 있습니다. → 【확인 필요】(부록 A, 교차 요구 확인).

## 결정 E. 배정판 3층 산출·무효화 (증분=신속, 일배치=정합성 진본)
- **결정**: 배정판을 ①수시 증분(계기 발생 시 당일) ②일 delta(날짜 롤오버분) ③주기 전체 재산출(정합성 진본) 3경로로 산출하되, **동일한 승자 확정 로직을 공유**합니다. 증분 누락·실패는 다음 전체 재산출이 보정합니다.
- **근거**: 증분은 지연 최소화, 전체 재산출은 정합성 보증이라는 역할 분담입니다(설계서 10장). 로직을 한 벌로 공유해 이원화 오류를 제거합니다.
- **대안**: 매일 전체 재산출만. → 수천만 행 이하에서는 유효하나 억 단위 도달 시 시간·비용이 커져 3층 전환이 필요합니다(조건부).
- **리스크**: 억~수십억 규모 전환 임계와 상위 규모 실측이 미확보입니다. → 【확인 필요】(부록 A-12).

## 결정 F. 캐시/정본 경계: 우대분만 배정, 미스는 정상 경로
- **결정**: 배정판에는 기본 대비 우대(이벤트·협의) 승자만 저장합니다. 저장되지 않은 셀(대다수)은 원장이 기본수수료를 직접 해석해 적용하며, 이는 예외가 아니라 정상 경로입니다.
- **근거**: 기본은 계좌마다 다르지 않아 계좌×셀 복제 저장은 낭비입니다. 우대분만 담아 저장 규모를 축소하고, 미스 경로는 전 계좌 공통·결정적입니다(설계서 1·6장).
- **대안**: 전 셀(기본 포함) 배정. → 억 단위 낭비이므로 기각.
- **리스크**: 정본(정책·요율표)과 파생(배정판)의 물리 배치·소유권·계약 버전 관리는 원장 스택 확인 후 확정됩니다. → 【확인 필요】(부록 A-07).

## 결정 G. 대규모 판의 우선순위 처리: 계산은 승인 때, 산출은 집합으로
- **결정**: 수십억 규모 판에서 우선순위 규칙 자체는 유지하되, 순위 계산 시점을 승인 시점으로 옮기고 산출을 집합 연산으로 수행합니다.
- **근거**: 체결 hot path에 계산을 두지 않고 사전 산출로 흡수한다는 원칙의 규모 확장입니다(설계서 10.4).
- **대안**: 체결 시 순위 실시간 계산. → hot path 비용·비결정성 유발로 기각.
- **리스크**: 집합 연산 산출의 물리 구현(스테이징·정렬 적재)은 구축 귀속이며 상위 규모 실측이 필요합니다. → 【확인 필요】(부록 A-12).

## 결정 H. 정산 시 플랫폼/배치 미가용 거동 (열화 모드)
- **결정**: **미정입니다.** → **【확인 필요】**(부록 A-05).
- **근거**: 체결 조회 미스 시 기본 요율 직접 해석 경로는 있으나, 정산 시점 플랫폼·배치 미가용 시의 열화 모드 정책(중단/지연/폴백 정산/사후 정정)은 저장소에 근거가 없습니다.
- **대안 / 리스크**: 열화 모드 정책이 확정되어야 가용성 목표(NFR-AVAIL-01)와 폴백 회계 처리 범위가 정해집니다. → 당사·현업·원장팀 협의 필요.
