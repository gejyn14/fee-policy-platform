# 수수료정책 과제 ISMP 산출물 — 개요 및 인덱스

- 과제: 차세대 원장 수수료 체계 재설계 (fee 도메인)
- 문서 성격: ISMP(정보화전략계획) 산출물 세트. fee 과제를 **참조 표본**으로 작성하며, 타 과제 산출물 작성 시 기준 템플릿으로 활용합니다.
- 기준 소스: 브랜치 `feature/v1-java-postgres` 최신 (기술설계서 **v1.7** + 실 구현 Java/Spring Boot 3 · PostgreSQL 16).
- 작성 기준일: 2026-07-08

## 0. 기준 버전 고지

- 본 세트는 기술설계서 최신판 **v1.7**(`docs/수수료정책플랫폼_기술설계서_v1.7.docx`)과 그에 대응하는 **실제 구현**(백엔드 `server/`, 프론트 `src/`)을 근거로 합니다.
- v1.7 판의 의미: 설계서상의 목표 모델(통합 랭킹·배정판·조회구분·3층 배치)이 **동작하는 코드로 구현·검증**되었습니다. 따라서 본 산출물의 요구·아키텍처 근거는 문서뿐 아니라 **참조 구현과 성능 실측**으로 뒷받침됩니다(부록: `05_부록_실현가능성증빙.md`).

## 1. 작성 범위 (상세화 기준)

- 본 세트의 상세화 수준은 **ISMP 수준**입니다 — 발주사·벤더가 입찰·검수하는 데 필요한 범위까지만 기재합니다.
- 그대로 코딩 가능한 수준의 상세(테이블 컬럼, DDL, API 스펙, 화면 레이아웃, 산출 알고리즘, 시퀀스, 사이징·샤드 수)는 구축 단계(SI) 과업으로, 본 산출물에는 기재하지 않습니다.
- 리포지토리의 상세설계서(v1.7)·물리 테이블 설계(`docs/table-design.md`)·동작 구현은 "실현 가능성 증빙"(부록 ⑤)으로만 인용하며, 그 상세를 본 산출물로 옮기지 않습니다. 벤더가 구현 방식을 결정할 여지를 남기기 위함입니다.

## 3. 표기 규칙

- 개조식·합쇼체로 기술하며, 주어는 당사(발주사) 또는 시스템입니다.
- **【확인 필요】**: 코드·설계서에 근거가 없어 당사·현업·컨설팅(KPMG)·원장팀이 확정해야 하는 항목입니다. 확정 전까지 임의 수치·가정을 기재하지 않고 비워 둡니다. 총괄은 부록 A(아래).
- 요구 식별자: 기능요구 `FR-{도메인}-{연번}`, 비기능요구 `NFR-{범주}-{연번}`. 기능분해·RFP가 이 ID를 교차 참조합니다.
- 검수 단계는 마크다운, 확정 후 외부 공유용 Word로 변환합니다.

## 부록 A. 미결 항목 (【확인 필요】)

코드·설계서에 근거가 없어 당사·현업·컨설팅·원장팀이 확정해야 하는 항목입니다. 본 세트는 이를 확정하지 않고 비워 두었습니다.

| ID | 항목 | 사유 | 필요 주체 |
|---|---|---|---|
| A-01 | 성능·규모 정량 목표 (계좌 수, 품목 수, 체결 피크 TPS, 응답시간 SLA, 데이터 보존기간) | 저장소는 규모의 질적 근거·로컬 실측만 보유, 실환경 목표 수치 없음 | 현업·원장팀 |
| A-02 | AS-IS 통제 수준 (현행 등급표 직조회·수작업 연장·엑셀 sum-up의 통제·감사·조직 현황) | 저장소는 To-Be 중심, 현행 통제 상세 없음 | 현업 |
| A-03 | 가용성·SLA 목표 및 장애 등급 정의 | 근거 없음 | 현업·원장팀 |
| A-04 | 교차 구간(가격대별 우열 역전) 정책 동시 지원 여부 | 설계상 지배관계 전제(전 구간 하회)만, 교차 구간은 미지원 | 컨설팅·현업 |
| A-05 | 정산 시 플랫폼 미가용 거동(열화 모드) 정책 | 체결 미스=BASE 직접 해석 경로는 있으나, 정산 열화 정책은 미정 | 현업·원장팀 |
| A-06 | 다중 인스턴스 캐시/랭킹 정합성(분산 무효화) 운영 정책 | 설계 스펙 범위 밖 | 원장팀·벤더 |
| A-07 | 원장 실제 기술 스택·DBMS 및 연계 방식(직접 쓰기 vs EAI) | 문서상 미확정(구현 Postgres·문서 Oracle은 기본값 가정) | 원장팀 |
| A-08 | 인증·권한·직무분리 정책, 개인정보·세율·환율 소스 정책 | 구현 스펙이 "인증/권한 범위 밖"으로 명시 | 정보보호·현업 |
| A-09 | 협의 요청 부분승인(계좌별 일부 승인) 지원 여부 | 요청번호 단위 일괄 승인만 구현 | 현업 |
| A-10 | 자격 정책 현업 설정(UI/운영) 범위 | 자산군별 기준이 참조 데이터·시드로 고정 | 현업 |
| A-11 | 데이터 이관·마이그레이션(현행 등급표→BASE 정책 이관, 계좌·종목 마스터 동기화 배치) 범위·절차 | 구현 스펙이 "마스터 동기화 배치·실 원장 연계 범위 밖"으로 명시 | 원장팀·벤더 |
| A-12 | 억 단위 상위 규모(10억 행+) 실측 및 3층 배치 전환 임계 | 로컬 실측은 2억 행까지, 그 이상은 B-tree 깊이 기반 추정 | 원장팀·벤더 |
